  <action>
Creer `_scripts/test-sync.js` — script Node.js zero dependency qui valide les artefacts generes par `sync-dictionnaire.js`. Le script utilise `process.exit(1)` en cas d'echec et `process.exit(0)` si tout passe.

**Tests a implementer (dans cet ordre) :**

1. **Test: Nombre de fichiers MD**
   - Lire le contenu de `_dictionnaire/` avec `fs.readdirSync`
   - Filtrer les fichiers `.md`
   - Verifier que le count est exactement 1408
   - Afficher : `PASS: 1408 fichiers MD generes` ou `FAIL: attendu 1408, obtenu {N}`

2. **Test: Front matter valide sur un echantillon**
   - Prendre 20 fichiers aleatoires dans `_dictionnaire/`
   - Pour chaque fichier : verifier qu'il commence par `---\n`, contient un second `---\n`, et que le front matter contient les cles obligatoires : `title`, `slug`, `permalink`, `category`, `letter`, `layout`
   - Verifier que `layout` vaut `definition`
   - Verifier que `permalink` suit le pattern `/dictionnaire/{slug}/`
   - Afficher : `PASS: front matter valide sur 20 echantillons` ou `FAIL: front matter invalide dans {fichier}: {raison}`

3. **Test: Pas d'UUIDs bruts dans les cross-references**
   - Regex UUID : `/[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}/`
   - Scanner les 20 memes fichiers echantillon
   - Pour chaque fichier : extraire la section front matter, chercher la regex UUID
   - Si un UUID brut est trouve dans le front matter (hors la cle `uuid` si elle existait) : FAIL
   - Afficher : `PASS: aucun UUID brut dans les cross-references` ou `FAIL: UUID brut trouve dans {fichier}`

4. **Test: Index JSON valide**
   - Lire `assets/data/search-index.json`
   - Parser avec `JSON.parse`
   - Verifier que c'est un tableau de longueur 1408
   - Verifier que chaque element a les 4 cles : `title`, `slug`, `category`, `letter`
   - Verifier que les `letter` sont des lettres majuscules A-Z
   - Afficher : `PASS: search-index.json valide (1408 entrees)` ou `FAIL: {raison}`

5. **Test: Images copiees**
   - Compter recursivement les fichiers `.png` dans `assets/img/dictionnaire/`
   - Verifier que le count est exactement 54
   - Afficher : `PASS: 54 images copiees` ou `FAIL: attendu 54, obtenu {N}`

6. **Test: Chemins d'images recrits**
   - Prendre les fichiers `_dictionnaire/adresse-de-reception.md` et tout autre fichier qui devrait avoir des images (trouver en verifiant la presence de `![` dans le contenu)
   - Verifier que les chemins d'images utilisent `/assets/img/dictionnaire/` et NON `./assets/`
   - Afficher : `PASS: chemins d'images recrits correctement` ou `FAIL: chemin non recrit dans {fichier}`

**Format de sortie :**
```
=== Validation post-sync ===
[1/6] PASS: 1408 fichiers MD generes
[2/6] PASS: front matter valide sur 20 echantillons
[3/6] PASS: aucun UUID brut dans les cross-references
[4/6] PASS: search-index.json valide (1408 entrees)
[5/6] PASS: 54 images copiees
[6/6] PASS: chemins d'images recrits correctement

=== 6/6 tests PASSED ===
```

Si un test echoue, afficher le detail et continuer les autres tests. A la fin, afficher le nombre de PASS/FAIL et quitter avec code 1 si au moins un FAIL.

**IMPORTANT :** Le script doit fonctionner sur Windows (paths avec `path.join`, pas de `/` en dur pour les chemins filesystem).
