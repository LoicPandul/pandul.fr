<task type="auto">
  <name>Task 2: Creer le script de sync complet avec resolution cross-refs, copie images, et index JSON</name>
  <files>_scripts/sync-dictionnaire.js</files>
  <read_first>_config.yml</read_first>
  <action>
Creer `_scripts/sync-dictionnaire.js` — script Node.js ZERO DEPENDENCY (uniquement `fs`, `path`, `process` natifs).

**Arguments CLI :**
- `--source PATH` : chemin vers le repo Dictionnaire (defaut: `../Dictionnaire`)
- `--dry-run` : affiche les stats sans ecrire de fichiers

**Structure du script (dans cet ordre) :**

1. **Parsing des arguments CLI** : lire `--source` et `--dry-run` depuis `process.argv`.

2. **Fonction `parseMetadata(content)`** : parser le YAML minimaliste du repo source. Le format a exactement ces cles possibles: uuid, title, slug, category, english_term, french_term, assets (liste de {filename}), cross_references (liste de strings UUID), version. NE PAS ecrire un parser YAML generique — parser uniquement ce format connu. Logique :
   - Iterer sur les lignes
   - Detecter les cles de niveau 0 avec regex `/^([a-z_]+):\s*(.*)$/`
   - Pour les valeurs simples (uuid, title, slug, category, english_term, french_term, version) : stocker la valeur en retirant les guillemets
   - Pour les listes (cross_references, assets) : collecter les lignes commencant par `  - ` jusqu'a la prochaine cle
   - Pour assets : extraire `filename:` de chaque item de liste
   - Retourner un objet JS avec toutes les cles

3. **Fonction `escapeYaml(str)`** : echapper les guillemets doubles dans les titres. Remplacer `"` par `\"` et `\` par `\\`. Les titres contiennent des MAJUSCULES, accents, tirets, points, apostrophes, parentheses.

4. **Fonction `rewriteImagePaths(content, slug)`** : remplacer toutes les occurrences de `![...](./assets/...)` par `![...](/assets/img/dictionnaire/{slug}/...)` avec la regex : `/!\[([^\]]*)\]\(\.\/assets\/([^)]+)\)/g`

5. **Fonction `generateFrontMatter(def)`** : generer le front matter YAML. Format exact de sortie (toutes les valeurs string entre guillemets doubles) :
```
---
title: "TITRE ECHAPE"
slug: "slug"
permalink: /dictionnaire/slug/
category: "CATEGORIE"
letter: "X"
layout: definition
english_term: "TERM"        # seulement si present
french_term: "TERM"         # seulement si present
cross_references:
  - title: "TITRE REF"
    slug: "slug-ref"
---
```
   La `letter` est `def.slug[0].toUpperCase()`. Les cross-references sont deja resolues (objets {title, slug}).

6. **Algorithme principal :**

   a. Scanner `{source}/definitions/fr/` — lister toutes les lettres (a-z), puis pour chaque lettre lister les sous-dossiers (slugs).
   
   b. **Passe 1 : construire la map UUID.** Pour chaque definition, lire `metadata.yaml`, parser, stocker dans `uuidMap = new Map()` avec cle=uuid, valeur={slug, title}. Aussi collecter toutes les definitions dans un tableau.
   
   c. **Passe 2 : resoudre les cross-references.** Pour chaque definition, remplacer la liste d'UUIDs `cross_references` par une liste d'objets `{title, slug}` en utilisant `uuidMap.get(uuid)`. Filtrer les UUIDs non resolus avec un `console.warn()`.
   
   d. **Passe 3 : generer les fichiers.** Pour chaque definition :
      - Lire `definition.md`
      - Appliquer `rewriteImagePaths(content, slug)` sur le contenu
      - Generer le front matter avec `generateFrontMatter(def)`
      - Concatener front matter + `\n\n` + contenu traite
      - Ecrire dans `_dictionnaire/{slug}.md` (forcer line ending `\n`, pas `\r\n`)
      - Si la definition a des assets : copier chaque fichier image de `{source}/definitions/fr/{letter}/{slug}/assets/{filename}` vers `assets/img/dictionnaire/{slug}/{filename}` (creer le dossier si necessaire)
   
   e. **Passe 4 : generer l'index de recherche.** Construire un tableau JSON avec pour chaque definition : `{title, slug, category, letter}`. Trier par `title` avec `localeCompare('fr')`. Ecrire dans `assets/data/search-index.json` (creer le dossier `assets/data/` si necessaire). Forcer line ending `\n`.

   f. **Afficher les stats en console :**
      - Nombre de definitions traitees
      - Nombre d'images copiees
      - Nombre de cross-references non resolues (warnings)
      - Taille du search-index.json en KB
      - Si `--dry-run` : afficher "DRY RUN — aucun fichier ecrit" et quitter apres la Passe 2

7. **Nettoyage avant ecriture :** Au debut de la Passe 3, supprimer le contenu de `_dictionnaire/` s'il existe (pour eviter les fichiers orphelins d'un run precedent). Creer le dossier s'il n'existe pas. NE PAS supprimer `assets/img/dictionnaire/` (les images sont idempotentes par copie).

8. **Gestion d'erreurs :** 
   - Si le dossier source n'existe pas : afficher un message clair avec le chemin tente et quitter avec code 1
   - Si un `metadata.yaml` est illisible : log un warning et continuer (ne pas crasher)
   - Si un UUID de cross-reference n'est pas dans la map : log un warning et ignorer cette ref

**IMPORTANT — Line endings :** Tous les `fs.writeFileSync` doivent utiliser des strings avec `\n` uniquement. Ne jamais utiliser `os.EOL`. Verifier que le contenu lu depuis les fichiers source est normalise avec `.replace(/\r\n/g, '\n')`.

**IMPORTANT — Encoding :** Tous les `fs.readFileSync` et `fs.writeFileSync` avec encoding `'utf8'`.
