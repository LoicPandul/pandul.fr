**Pass 5 -- Compute prev/next within each category and add category_slug to front matter:**

Add a `slugify` function (reuse for all slug generation):
```javascript
function slugify(str) {
  return str.normalize('NFD').replace(/[\u0300-\u036f]/g, '').toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '');
}
```

Group definitions by category, sort alphabetically within each group by title (using `localeCompare('fr')`), then inject into each definition object:
- `category_slug`: slugify(def.category)
- `prev_in_category`: { title, slug } of the previous definition in same category (null if first)
- `next_in_category`: { title, slug } of the next definition in same category (null if last)

Update `generateFrontMatter` to emit these new fields:
```yaml
category_slug: "organisation"
prev_in_category:
  title: "Previous Term"
  slug: "previous-term"
next_in_category:
  title: "Next Term"
  slug: "next-term"
```

If prev or next is null, omit the field entirely.

**Pass 6 -- Generate category pages in _categorie/:**

Create directory `_categorie/` (clean it first like _dictionnaire/ is cleaned).

For each of the 19 categories, generate a markdown file `_categorie/{category_slug}.md` with YAML front matter containing these fields:
- title: "{Category Name}" (the raw category string, e.g. "PROTOCOLE")
- slug: "{category_slug}" (slugified, e.g. "protocole")
- permalink: /dictionnaire/categorie/{category_slug}/
- layout: category
- definition_count: {N} (integer, number of definitions in this category)
- letters_with_definitions: ["A", "B", "C", ...] (sorted array of uppercase letters that have at least one definition in this category)

`letters_with_definitions` is used by the A-Z bar on category pages to gray out empty letters.

**Pass 7 -- Generate letter pages in _lettre/:**

Create directory `_lettre/` (clean it first).

For each letter A-Z (all 26), generate `_lettre/{letter}.md` with YAML front matter containing:
- title: "Lettre {X}" (e.g. "Lettre A")
- slug: "{x}" (lowercase, e.g. "a")
- permalink: /dictionnaire/lettre/{x}/ (lowercase)
- layout: letter
- letter: "{X}" (uppercase, e.g. "A")
- definition_count: {N} (integer, 0 if no definitions for this letter)

If a letter has 0 definitions, still generate the page (definition_count: 0).

Log stats for passes 5-7 to console, matching existing log style.

6. Definition list: `<div class="dict-list">` using `{% assign sorted_defs = site.dictionnaire | where: "category", page.title | sort: "title" %}`.
